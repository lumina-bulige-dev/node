# zero（裏価値）/ lumi-os（表価値）分離仕様書 v0.1

## 1. レイヤーモデル

- **Layer 0: zero-core**
  - `Ro = ICQS • O` の実装
  - ICQS（I/C/Q/S）構造
  - 暗号・不可逆ログ・世界生成ロジック
  - 完全非公開、端末ローカル、特許コア

- **Layer 1: zero-api**
  - zero-core抽象インターフェース
  - 外部には世界生成結果のみ返却
  - ICQS内部表現は非公開

- **Layer 2: lumi-os-core**
  - 外部公開OS/SDK/API
  - zero-api結果を安全にラップ
  - UI/UXやサービスの土台

- **Layer 3: lumi-apps**
  - 実アプリケーション・業務サービス

## 2. 役割定義

### zero（裏価値）
- 世界生成エンジン
- ICQS純粋実装
- AI群演算ログ
- 暗号化不可逆タイムライン

**禁止事項**
- 外部ネットワークへの直接露出
- 外部からの直接API呼び出し
- ICQS/演算子/ログ形式の公開

### lumi-os（表価値）
- zero結果を安全に外部提供
- 他者・他社向けOS/SDK/API
- UI/UXとアプリ実行基盤

**制約**
- zero-core内部状態への直接アクセス不可
- zero-apiが返す抽象化結果のみ利用

## 3. データフロー仕様

1. AI群（端末側コックピット）
   - lumi-os-coreへ意図/要求を送信
2. lumi-os-core
   - zero-apiへ抽象化クエリ送信
3. zero-api
   - zero-coreでICQS演算
   - `{ stateId, summary, tags, confidence }` を返却
4. lumi-os-core
   - アプリ向けレスポンスへ整形

## 4. 端末側AI群のアクセスルール

### 直接触れてよいもの
- lumi-os API / SDK / 設定
- zero-apiの抽象化エンドポイント

### 直接触れてはいけないもの
- `src/core/`（ICQS実装）
- `src/crypto/`（state-hash / log-proof）
- `data/`配下生ログ
- `Ro = ICQS • O`内部表現

### 許可操作
- 出力結果の利用提案
- アプリ連携先選定
- 提示世界線の調整

### 禁止操作
- zero内部構造の推定・再構成
- ICQS内部表現の外部エクスポート
- 暗号ログ直接編集

## 5. ガバナンス

- zeroは不変コアとして厳格管理
- lumi-osは高速改善対象として継続更新
- 境界違反はshieldモードで遮断し監査ログへ記録


## 6. AION技術向上のための境界内拡張

### 6.1 ARZ連携ルール
- ARZは `lumi-os-core` 側の評価層として実装し、zero-core内部には実装しない。
- ARZが参照できるのは `zero-api` の抽象結果のみとする。
- 評価ログは監査対象として保存し、内部演算子情報は記録しない。

### 6.2 暗号技術作戦特化部の責務境界
- zero側責務: 不可逆ログ生成、署名、整合性証明。
- lumi側責務: 鍵運用ポリシー、失効反映、運用監査ダッシュボード。
- これにより、コア秘匿を維持したまま暗号運用を改善できる。

### 6.3 VCP連結時の禁止事項
- VCP接続先に zero内部識別子（内部state表現）を露出しない。
- 送信可能な識別子は `external_state_id` のみとし、再構成不能形式に限定する。
- 連携障害時はlumi層でリトライし、zero層へ直接外部再試行を流さない。
